qPCR Visualizer

A JS visualization library built on D3

The qPCR Visualizer is a JS library that allows you load large JSON datasets representing PCR trays and display them with your choice of a few modules based on your use case. The modules are largely configurable and allow for interaction between each other. Listed below are the various modules in their default state (with no styling except for the detail table module) with a basic description, and you can visit the GitHub page to get the code and view more in-depth documentation, and be sure to check out the various example configurations for a sense of how this library can be used.

  • Graph Module
    expand for more info

    The graph module shows fluorescence vs cycle count for all sample cells. It can be configured to have an info box, a fluorescence threshold line, different color sets and grouping options within those sets, custom sizing, and axis labels. The only requirement is a container element. The graph interacts with the grid module to allow for highlighting of row or column groups as well as individual cells. It exposes the fluor-chart class on its top-level svg element.

  • Grid Module
    expand for more info

    The grid module shows all of the sample cells from the qPCR tray. It can be configured with an info box, custom sizing, and custom coloring for the cells and headers. The grid interacts with the graph to highlight hovered elements, and also interacts with the detail module to show the exact fluorescence values for a specified sample cell. It exposes the fluor-grid class on its top-level svg element.

  • Detail Module
    expand for more info

    The detail module shows the fluorescence data for a specific cell. It's only configuration option is the addition of a threshold concept, but it is html (not svg) and thus can be manipulated with styling. It exposes the fluor-table class on its top-level table element, as well as the fluor-detail class on its text-containing span elements. There is an example CSS file for styling the table in the github repo.

Global configuration options

There are a few options that are configured at the global level:

  • URI of the target JSON data
  • Options for parsing the passed in JSON object (such as a key regex and data point names)
  • Info box presence
  • Fluorescence threshold behavior

Future developments

There are a couple of features I'd like to implement in the future:

  • Better mobile-friendliness and responsiveness
  • Research mode toggle for grid - would allow selection and focus-locking of random selection of cells
  • Dynamically change grouping mode for coloring graph
  • Dynamically change dataset
  • Annotate data set with reagents and other notes and re-export
  • Toggle default inline styles on and off
